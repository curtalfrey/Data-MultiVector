Data-MultiVector Installation Instructions

Deploying Data-MultiVector using Docker

  1. Clone the Repository:
    • Open a terminal on your server and navigate to your desired deployment directory.
    • Run the following command to clone the repository: git clone https://github.com/curtalfrey/Data-MultiVector.git /path/to/deployment/directory
  2. Navigate to the Project Directory:
    • Change into the cloned repository directory:
    • Build the Docker image: cd /path/to/deployment/directory
      docker-compose build
  3. Configure Environment Variables:
    • Open the docker-compose.yml file and update the environment variables under the app service with your desired values.
    • Adjust the INPUT_PATHS and OUTPUT_PATHS according to the directories you want to link.
  4. Build and Start Docker Containers:
    • Run the following command to build and start the Docker containers defined in the docker-compose.yml file:
    • Start the application in detached mode: docker-compose up -d
  5. Access the Application:
    • Open a web browser and navigate to http://localhost:5000 to access Data-MultiVector.
    • If you've configured a subdomain, set up the domain mapping accordingly.
  6. Provide Feedback and Report Bugs:
    • If you encounter issues or bugs, please report them on the GitHub repository: https://github.com/curtalfrey/Data-MultiVector/issues. Your feedback helps improve the application.
