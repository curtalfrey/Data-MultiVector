Developer Instructions - data-multivector Project

1. Data Processing Logic

Implement the logic to process input paths, output paths, and connections:

  • Create dictionaries or data structures to store connection data, change logs, and user trash data.
  • Enhance the connection_data dictionary to include relevant data for each connection, such as processing status, timestamps, etc.
  • Implement logic for user authentication, registration, and management.
  • Implement logic to handle data storage, such as creating symlinks, copying files, and user trash management.
  • Ensure data validation and security checks to prevent malicious input.

2. Logging Configuration

Enhance logging functionality for better debugging and monitoring:

  • Add timestamps and log levels to log entries.
  • Improve log messages with relevant information about each logged event.

3. Error Handling

Implement error handling to gracefully handle exceptions:

  • Wrap relevant parts of the code in try-except blocks to catch and handle exceptions.
  • Provide meaningful error messages to users and log details for debugging.

4. Security Measures

Ensure the security of the application:

  • Implement CSRF protection to prevent cross-site request forgery attacks.
  • Secure routes and endpoints to restrict access to authorized users.
  • Validate and sanitize user input to prevent injection and other security vulnerabilities.

5. Frontend Design

Design user-friendly frontend templates:

  • Create or update HTML templates (index.html, register.html, login.html, dashboard.html) for better user interaction.
  • Add forms, input fields, and buttons for submitting data to the backend.

6. Deployment Considerations

Prepare the application for production deployment:

  • Replace the Flask development server with a production-ready server (e.g., Gunicorn).
  • Configure the application for secure deployment using HTTPS.
  • Integrate Docker and Docker Compose for containerized deployment.
